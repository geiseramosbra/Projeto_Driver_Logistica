PROJETO DRIVER: LOGISTICS ANALYSIS AND SUPPLY CHAIN PIPELINE
ENGLISH VERSION
PROJECT OVERVIEW (STAR METHOD)
SITUATION
The logistics department required a robust way to monitor delivery performance and identify risks across different markets, specifically LATAM and USCA. The raw data contained over 180,000 records with inconsistent naming and unnecessary columns for the final business analysis.

TASK
The objective was to build an automated ETL (Extract, Transform, Load) pipeline using Python to clean the dataset, calculate logistics delay metrics, and create an interactive dashboard for executive decision-making.

ACTION

Developed a Python script (pipeline_draiver.py) using Pandas for data manipulation.

Implemented filtering logic to focus on strategic markets and specific product categories.

Created a calculated feature to measure the difference between scheduled and real shipping days to quantify delays.

Designed a Power BI dashboard focused on Late Delivery Risk and Market Health.

RESULT

Successfully processed 77,393 lines of logistics data after cleaning.

Automated the generation of a refined CSV, reducing manual data preparation time.

Built a visual map and risk analysis charts that identify delivery gaps by city and category.

TECH STACK
Python (Pandas)

Power BI

Git/GitHub

SITUAÇÃO
O departamento de logística precisava de uma forma robusta para monitorar o desempenho de entrega e identificar riscos em diferentes mercados (LATAM e USCA). Os dados brutos continham mais de 180.000 registros com nomes inconsistentes e colunas desnecessárias para a análise de negócio final.

TAREFA
O objetivo era construir um pipeline de ETL (Extração, Transformação e Carga) automatizado usando Python para limpar o conjunto de dados, calcular métricas de atraso logístico e criar um dashboard interativo para tomada de decisão executiva.

AÇÃO

Desenvolvi um script Python (pipeline_draiver.py) utilizando Pandas para manipulação de dados.

Implementei lógica de filtragem para focar em mercados estratégicos e categorias de produtos específicas.

Criei uma coluna calculada para medir a diferença entre os dias de envio agendados e reais para quantificar atrasos.

Desenvolvi um dashboard no Power BI focado em Risco de Entrega Tardia e Saúde do Mercado.

RESULTADO

Processamento bem-sucedido de 77.393 linhas de dados logísticos após a limpeza.

Automação da geração de um CSV refinado, reduzindo o tempo de preparação manual de dados.

Construção de um mapa visual e gráficos de análise de risco que identificam gargalos de entrega por cidade e categoria.
